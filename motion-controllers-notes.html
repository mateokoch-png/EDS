1. PID basics

The error e = desired position − actual position. Three terms act on that error in parallel:

P (proportional): output = Kp × e. Bigger error → bigger correction, right now. Fast, but a P-only controller settles with a small leftover error under load (it needs SOME error to keep producing a correcting output).
I (integral): output = Ki × ∫e dt — it keeps accumulating as long as ANY error remains, even a tiny one. That's exactly why it eliminates steady-state error: a P-only controller can sit forever at a small nonzero error, but the integral term keeps growing for as long as that error exists, pushing the output further until the error reaches exactly zero.
D (derivative): output = Kd × de/dt — reacts to how fast the error is changing, acting as a brake when the system is approaching the target fast (reduces overshoot).

Why "a well-tuned PID controller achieves zero steady-state error" is TRUE (for a constant/step target): as long as the system is stable, the integral term will keep growing for as long as ANY error persists. The only way the integral term can stop growing is if e finally reaches exactly zero. So a stable PID loop with integral action is mathematically guaranteed to drive constant-target error to zero eventually — that's the whole reason the I term exists.

2. Feedforward — helps tracking, doesn't guarantee stability

What it is: instead of waiting for an error to appear and then reacting to it (like P/I/D do), feedforward adds a term computed directly from the KNOWN desired motion — e.g. velocity feedforward adds Kvff × (desired velocity) straight into the output, since you already know roughly how much drive signal is needed to achieve that speed, without needing an error to build up first.

This reduces "following error" (the lag between commanded and actual position during motion) because the controller isn't purely reactive anymore — part of the output is pre-computed from the plan itself.

Why "feedforward guarantees stability regardless of PID tuning" is FALSE: feedforward only adds an extra, open-loop, pre-computed signal on top of the existing PID loop. It does nothing to change the feedback loop's own dynamics — if your Kp/Ki/Kd combination is unstable (e.g. way too high gain, causing oscillation that grows instead of shrinks), adding feedforward doesn't fix that instability at all. Feedforward improves TRACKING ACCURACY, not STABILITY — these are two separate properties of a control loop, and exams like to test whether you can tell them apart.

3. Velocity profiles — why parabolic isn't time-optimal

A trapezoidal profile (constant accel → constant cruise speed → constant decel) reaches and HOLDS the maximum allowed velocity for as long as possible — that flat top is where distance gets covered fastest.

A parabolic profile never has a flat-top cruise phase at all — velocity is constantly changing (curving) the whole time, meaning it spends LESS time at or near maximum velocity for a given peak acceleration. Less time at high speed, for the same move distance and the same acceleration limit, means MORE total time — parabolic is slower, not faster.

This is why "parabolic is optimal/fastest for point-to-point motion" is FALSE — trapezoidal is faster for a given vmax/amax limit. (A parabolic-style smoothing IS used when you need to limit JERK, like an S-curve, but that trades away time-optimality on purpose, for smoothness — it's not claiming to be the fastest option.)

4. The exact recurring statements (verbatim pattern across exam years)

Recurring T/F pair — appears in 2023 + both 2024 exams

Statement 1: "A parabolic velocity profile is optimal/fastest for point-to-point motion." → FALSE — trapezoidal is faster for the same vmax/amax (see section 3).

Statement 2: "A PM-DC motor still heats up during regenerative braking." → TRUE — I²R copper losses still occur regardless of energy direction (covered in the Motor Control notes).

Answer pattern: only statement 2 is correct.

Recurring T/F pair — appears in 2023 + both 2024 exams

Statement 1: "A well-tuned PID controller eliminates steady-state error for a step input, provided the system is stable." → TRUE — the integral term guarantees this (see section 1).

Statement 2: "Adding feedforward guarantees stability regardless of how the PID gains are tuned." → FALSE — feedforward improves tracking, not stability (see section 2).

Answer pattern: only statement 1 is correct.

If reworded

Watch for a version that swaps which statement is true/false, or changes "PM-DC motor" to a different motor type, or changes "parabolic" to "S-curve" (S-curve IS smoother/lower-jerk than trapezoidal, but still not faster — don't let a reworded version trick you into thinking smoother = faster).
